## 2. Technology Stack

| Layer | Technology | Rationale |
|-------|-----------|-----------|
| **Frontend** | Next.js 15, TypeScript, Tailwind CSS | SSR for SEO, type safety, utility-first CSS |
| **Backend** | FastAPI (Python 3.12), Uvicorn | Async performance, Pydantic validation, AI ecosystem |
| **Primary DB** | PostgreSQL 16 | Relational data, JSON support, full-text search |
| **Cache** | Redis 7 | Session store, rate limiting, response cache |
| **Vector DB** | Qdrant | Curriculum RAG, semantic search |
| **AI Orchestration** | LangGraph | Multi-agent state machines, tool calling |
| **Reverse Proxy** | Nginx | TLS termination, static serving, load balancing |
| **Containerization** | Docker + Docker Compose | Consistent dev/prod environments |
| **Monitoring** | Langfuse + Prometheus + Grafana | LLM tracing + system metrics |
| **CI/CD** | GitHub Actions | Automated testing + deployment |
| **TTS** | ElevenLabs / Azure Cognitive Services | Natural voice for EN/HI/BN |
| **STT** | OpenAI Whisper | Best accuracy for Indian languages |
| **Math Engine** | SymPy | Deterministic math verification |
| **Auth** | JWT + OAuth2 (Google) | Standard, secure authentication |

## 8. Scaling Strategy

### Phase 1 (MVP вҖ” 1K concurrent)
- Single Docker host, all services on one machine
- PostgreSQL with connection pooling (PgBouncer)
- Redis for session + cache
- 4 Uvicorn workers

### Phase 2 (Growth вҖ” 10K concurrent)
- Separate DB to managed PostgreSQL (AWS RDS / Azure Postgres)
- Redis Cluster for session distribution
- Multiple FastAPI instances behind load balancer
- CDN (Cloudflare) for static assets
- Read replicas for analytics queries

### Phase 3 (Scale вҖ” 100K+ concurrent)
- Kubernetes (AKS/EKS)
- Horizontal pod autoscaling
- Message queue (RabbitMQ) for async report generation
- Multi-region deployment
- Sharded database by region
